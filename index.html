Google Maps API

near app

I did this project to explore the Google Maps API, and to see how React would work with a complex embedded object (a google map.)

The idea is pretty simple. You set your location (if you're not happy with the auto-locator), choose what you're after and how far away, and press search. A random place matching your criteria will popup, from which you can opt for direcitons, or search again.

near app search result

Normally you'd have the map as the highest level component, and pass its data down to the sub-components for rendering (to follow the 'React' way.) Though we don't want the map to reload each time the data changes.

Instead, a messaging system was brought up to the highest level, and was used to send data between the Map component, and the other components. Implementing this really showed me the boundaries of react, and the need for such architectures as Flux and Redux.

For now I'm happy using this app as a learning experience. Though perhaps one day I'll expand on the idea and see if I can commercialize it.
